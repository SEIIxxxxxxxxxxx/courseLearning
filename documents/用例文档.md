## 1. 引言

### 1.1 目的

本文档描述了课程学习网站 **courselearning** 的用户需求（第三阶段），为后期开发人员对系统的实现和验证工作提供指导，且面向开发人员、测试人员及最终用户而编写，是了解系统的导航。

### 1.2 阅读说明

用例描述的约定详见下表所示：

| 下文出现名词 |           对应全称或具体含义           |
| :----------: | :------------------------------------: |
|     系统     |       courselearning课程学习系统       |
|     用户     | 在不区分老师和学生的情况下对两者的统称 |

## 4. 详细用例描述

### 4.1 租用课程

| ID                 | 1                                                            |
| ------------------ | ------------------------------------------------------------ |
| **名称**           | 租用课程                                                     |
| **创建者**         | 石善羽                                                       |
| **创建日期**       | 2021.6.11                                                    |
| **最后一次更新者** | 石善羽                                                       |
| **最后更新日期**   | 2021.6.13                                                    |
| **参与者**         | 学生，目标是在一段时间内获得指定课程的查看权                 |
| **触发条件**       | \                                                            |
| **前置条件**       | 学生必须已注册且已登录合法账号                               |
| **后置条件**       | 系统将生成课程订单信息，且学生能够查看相应课程内容           |
| **优先级**         | 高                                                           |
| **正常流程**       | 1. 学生对指定课程发起租用课程请求<br />2. 系统跳转至租用课程界面<br />3. 学生发起选择租用方案请求<br />4. 系统跳转至租用方案选择界面<br />5. 学生选择租用时长（7天，15天或30天），并确认此租用方案<br />6. 系统展示租用金额，生成租用课程订单，订单信息包括租用价格、课程id、课程名、创建时间、状态信息、起始日期、截止日期，并将此订单添加至订单列表，跳转至租用支付确认界面<br />7. 学生确定并支付租用费用<br />8. 系统更新订单状态为已支付，更新课程状态为已购买，更新余额，跳转至课程展示界面 |
| **扩展流程**       | 2a. 系统检测出学生已购买该课程：<br />       1. 系统拒绝生成租用订单，并跳转至课程展示界面<br />1-5c.  学生发起取消租用请求：<br />       1. 系统跳转回课程信息界面<br />7a. 学生发起取消支付请求：<br/>       1. 系统跳转至订单列表界面<br />8a. 学生账户余额不足：<br />       1. 系统提示余额不足，并跳转至充值界面 |
| **特殊需求**       | 1. 若系统时间超过租用课程的截止时间，课程将回复至锁定状态，同时更新租用订单信息 |

### 4.2 开通会员

|         ID         | 2                                                            |
| :----------------: | :----------------------------------------------------------- |
|      **名称**      | 开通会员                                                     |
|     **创建者**     | 黄昱霖                                                       |
|    **创建日期**    | 2021.6.10                                                    |
| **最后一次更新者** | 黄昱霖                                                       |
|  **最后更新日期**  | 2021.6.13                                                    |
|     **参与者**     | 学生，目标是在一段时间内获得所有课程的查看权                 |
|    **触发条件**    | \                                                            |
|    **前置条件**    | 学生必须已注册且已登录合法账号                               |
|    **后置条件**    | 系统解锁后再次收回学生对所有课程的查看权                     |
|     **优先级**     | 中                                                           |
|    **正常流程**    | 1. 学生发起开通平台会员请求<br />2. 系统跳转至购买会员界面<br />3. 学生选择会员卡类型（日卡、月卡或年卡），并确认购买<br />4. 系统生成购买会员的订单，内容包括学生id，会员有效期时长与会员到期时间，并更新所有课程的状态为已购买，更新用户级别为尊贵会员，更新账户余额，跳转至个人信息界面<br />5. 会员到期后，系统将所有课程状态设置为未购买 |
|    **扩展流程**    | 3a. 学生取消开通平台会员请求：<br />      1. 系统跳转回个人信息界面<br />4a. 学生账户余额不足：<br />      1. 系统提示错误，提醒学生充值，并跳转回个人信息界面<br />4b. 学生已经开通过会员：<br />      1. 系统在原定会员到期时间上，增加新购买会员对应的会员有效期时长 |
|    **特殊需求**    | 1. 会员到期时间的格式为xxxx-xx-xx xx:xx:xx<br />2. 若学生在开通会员前已经购买了部分课程，则开通会员及会员到期不会对这些课程产生任何影响 |

### 4.3 创建题目

|         ID         | 3                                                            |
| :----------------: | :----------------------------------------------------------- |
|      **名称**      | 创建题目                                                     |
|     **创建者**     | 钟山                                                         |
|    **创建日期**    | 2021.6.10                                                    |
| **最后一次更新者** | 钟山                                                         |
|  **最后更新日期**  | 2021.6.13                                                    |
|     **参与者**     | 教师，目标是创建题目并添加到题库                             |
|    **触发条件**    | \                                                            |
|    **前置条件**    | 教师必须已注册且已登录合法账号                               |
|    **后置条件**    | 系统存储创建的题目，跳转至课程管理界面                       |
|     **优先级**     | 中                                                           |
|    **正常流程**    | 1. 教师发起题目创建请求<br />2. 系统跳转至题目创建界面<br /><br />3.a. 教师选择创建单选题<br />4.a. 系统跳转至单选题的创建界面<br />5.a. 教师添加题干和选项，设置一个选项为正确答案，添加解析，选择所属课程，确定创建题目<br />3.b. 教师选择创建多选题<br />4.b. 系统跳转至多选题的创建界面<br />5.b. 教师添加题干和选项，设置若干个选项为正确答案，添加解析，选择所属课程，确定创建题目<br />3.c. 教师选择创建论述题<br />4.c. 系统跳转至论述题的创建界面<br />5.c. 教师添加题干、正确答案，添加解析，选择所属课程，确定创建题目<br /><br />6. 系统存储题目信息，跳转至课程管理界面 |
|    **扩展流程**    | 1a. 教师未创建任何课程：<br />      1. 系统提示错误并跳转至课程管理界面<br />5.a/b/c.a. 教师未输入题干：<br />      1. 系统提示错误并提示输入题干<br />5.a/b.b. 教师未添加选项：<br />      1. 系统提示错误并提示添加选项<br />5.a/b.c. 教师未设置正确选项：<br />      1. 系统提示错误并提示选择正确选项<br />5.c.b. 教师未输入答案：<br />      1. 系统提示错误并提示输入答案<br />5.a/b/c.c 教师未选择所属课程：<br />      1. 系统提示错误并提示选择所属课程<br />3-5a. 教师选择取消创建题目：<br />      1. 系统跳转至课程管理界面 |
|    **特殊需求**    | 1. 单选题设置第二个正确选项，则第一个正确选项自动取消<br />2. 题干的字数上限为150字符<br />3. 选择题选项的字数上限为50字符<br />4. 论述题答案的字数上限为300字符 |

|    **扩展流程**    | 1a. 教师未创建任何课程：<br />      1. 系统提示错误并跳转至课程管理界面<br />1b. 教师未创建任何题目：<br />      1. 系统提示错误并跳转至课程管理界面<br />3a. 教师未选择任何题目：<br />      1. 系统提示错误并提示选择题目<br />3b. 教师设置开始时间晚于结束时间：<br />      1. 系统提示错误并提示重新设置时间<br />3c. 教师选择取消创建测试：<br />      1. 系统跳转至课程管理界面 |
